# Text Editors Lab

## Objective
1. Use `wget` to download a file from the terminal
2. Use a command-line text editor to edit files

## Vocabulary
| Vocabulary| Description |
| ----------- | ----------- |
| in-line text editor  | A program that edits text files from within the terminal |
| `nano <filename>`  | A program to edit files from the terminal |
| `vi <filename>`  | A program to edit files from the terminal |
| `wget <link>`  | A program that downloads files |

## Your Task

Your task is to download a text file and edit it using an in-line text editor. 

## Getting Started

1. Use wget to download [Pride and Prejudice](https://www.gutenberg.org/ebooks/1342) as a .txt file from Project Gutenberg. You want the UTF-8 version.
2. Learn how to use the following features with `nano` or `vi`(more challenging)
    1. Open a file
    2. Save and exit
    3. Copy, cut, and paste
    4. Go to the beginning of a file
    5. Go to the end of a file
    6. Search for a target string
3. Create a PDF to document how to do each of these things. Think about how you can organize this information most effectively - like with a table, screenshot, or code blocks.

## Extensions
1. Create a custom keybinding for vim by editing your `.vimrc` file. You can make whatever you want, but some ideas are to save and exit with CTRL-S or allow paste with CTRL-P

## Reflection Questions
1. What is a text editor?
2. Why is important to learn a text editor?
3. Give an example where a mouse won't be available and terminal will be the only option you have.
