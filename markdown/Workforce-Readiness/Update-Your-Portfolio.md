# Update Your Portfolio

<img src="/assets/portfolio-header.png">

1. Create a README for your account to introduce yourself.
    1. Create a repository that shares the same name as your username. Make sure you have the exact same spelling.
    2. In the repository, create a file called “README.md”. The contents of this file will appear on your profile automatically. Try adding a header with your name and clicking on your profile. 
    3. Add a few things to your README. For example you could add
        - An introduction or header image
        - Something you are currently working on, learning, or curious about
        - The technology you are most comfortable with(programming languages, operating systems, etc.). Icons work great here. You can use [simpleicons.org](https://simpleicons.org/) or [shields.io](https://shields.io/) to generate your own. 
        - Add your [GitHub stats](https://github.com/anuraghazra/github-readme-stats) or other live updating information
    4. See [these templates](https://github.com/durgeshsamariya/awesome-github-profile-readme-templates/tree/master/templates) and [these templates](https://recodehive.github.io/awesome-github-profiles/) for inspiration. Remember that the source code for these are all public so you can just copy it directly!
2. Pin a few of your most important projects to your profile.
    1. Make sure these repos have good quality README’s that explain your project. You want users to be able to understand what you created, how it works, and how to use it. 
3. Private any incomplete repositories.
    1. Private anything incomplete or projects you don’t want to appear on your public portfolio. As you continue in your career, you will probably want to private older projects that no longer represent your skills.
4. Circle where you are at in each category of [the portfolio rubric](https://docs.google.com/document/d/13dplZyEPZFKyoX3lDOXnQc8E9R4UTlDmR2PUlCWq_qs/edit?usp=sharing) and turn it in. 
    1. If you are in your first year, you should be aiming for the Approaching category. If you are in your second year, you should be aiming for the Proficient category or above.
5. Extra Credit: Create a portfolio website
    1. Create a new repository with the name “yourusername.github.io” 
    2. Add a file to it called “index.html” with a few lines of text in it. Then put “yourusername.github.io” in your browser to access your site. You should see your lines of text appear! This site can serve as a custom portfolio and will be accessible by anyone online. 
    3. Program your own site(or vibe code it) and add information about you. You’ll want to highlight your most significant projects and experiences.
