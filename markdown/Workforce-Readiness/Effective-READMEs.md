# Applying Technical Writing to Create Effective READMEs

## Objective

1. Leverage principles of effective technical writing to create professional README documentation
2. Give effective feedback to peers about their work

## Getting Started

1. Review these [Notes On Technical Writing](#Workforce-Readiness/Technical-Writing-Notes)
2. Review [this README](https://github.com/emalderson/ThePhish#readme) and answer the following questions:
    - What information was conveyed with this README?
    - What do you notice about the writing style of this README? 
    - How did this README effectively or ineffectively use visuals and formatting?
    - What qualities do you see in this README that you want to bring to your own?


## Your Task 

1. Choose a GitHub project you would like to create a README for. The best choice would be something you want to highlight on your resume in the long term. 
2. Once you have your project cloned with `git clone <url>`, add a new file `README.md`. Practice pushing your code with:
```bash
git add .
git commit -m "message"
git push -u origin main
```

In the future, you can update your README with:

```bash
git add .
git commit -m "message"
git push 
```
3. You should see `README.md` appear in your online repository.
4. Review [Writing in Markdown Notes](#Workforce-Readiness/Writing-in-Markdown-Notes) for formatting strategies
5. Add the following sections to your README document and push your code
    - Title  
    - Description
    - Installation
    - Usage
    - Resources and Citations
6. Use these [examples of quality READMEs](https://github.com/matiassingers/awesome-readme?tab=readme-ov-file) to help your develop your own.

## Getting Feedback

The last part of this assignment is to get some feedback about your documentation. Swap your documentation with a peer and have them [complete this rubric with comments](https://docs.google.com/document/d/1LrhdraPLX-lXz2Ku8lUqZUesXqpxN3zsBNx_LR8l9sg/edit?usp=sharing). Attach the rubric to your assignment.  

Then, answer the following questions yourself.

1. What was my highest-scoring category, and what specific choices did I make that led to that success?
2. What was my lowest-scoring category? Does this align with what I felt was the weakest part of my draft?
3. Based on my reflections above, what are the top two specific goals I have for my next technical writing assignment? (e.g., "I will use bullet points for all instructions," or "I will remove all passive voice.")
4. What is my plan for proofreading next time? (e.g., "I will finish the draft 24 hours early so I can review it with fresh eyes," or "I will have a peer review it.")


## Rubric

- **30 points** - Created a well-written, professional, and informative README with appropriate visuals and formatting. Completed reflection questions thoroughly and professionally. Complete peer rubric with action steps. 
- **15 points** - Create a README but is informal, poorly formatted, or otherwise incomplete. Reflection answers are shallow or incomplete. Peer rubric has been completed, but does not have strong actionable steps identified. 
- **0 points** - Missing or incomplete
