# Applying Technical Writing to Create Effective READMEs

## Objective

1. Leverage principles of effective technical writing to create professional README documentation

## Getting Started

1. Choose a GitHub project you would like to create a README for. The best choice would be something you want to highlight on your resume in the long term. 
2. Once you have your project cloned with `git clone <url>`, add a new file `README.md`. Practice pushing your code with:
```bash
git add .
git commit -m "message"
git push -u origin main
```
3. You should see `README.md` appear in your online repository.
4. Review [Writing in Markdown Notes](#Workforce-Readiness/Writing-in-Markdown-Notes) for formatting strategies
5. Add the following sections to your README
    - Title  
    - Description
    - Installation
    - Usage
    - Resources and Citations
